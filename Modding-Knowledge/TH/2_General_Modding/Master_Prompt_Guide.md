# 🤖 Master Prompt: เทคนิคการเขียนคำสั่งควบคุม AI ผู้ช่วยนักแปล

ไฟล์นี้คือเทมเพลตคำสั่ง (System Prompt) ระดับมืออาชีพที่ถูกออกแบบมาเพื่อใช้ควบคุม AI (เช่น ChatGPT, Gemini, Claude) ให้ทำงานเป็นผู้ช่วยในโปรเจกต์ม็อดแปลภาษาเกมอย่างเป็นระบบ โดยป้องกันไม่ให้ AI ทำงานข้ามขั้นตอนหรือทำไฟล์พัง

> [!TIP] วิธีใช้งาน
> ให้คุณคัดลอกข้อความในกล่อง **โค้ดด้านล่าง** ทั้งหมด นำไปแก้ข้อมูลในวงเล็บ `[...]` ให้ตรงกับเกมที่คุณกำลังทำอยู่ จากนั้นส่งให้ AI เป็นข้อความแรกสุดของการเริ่มต้นแชทใหม่

---

## 📋 Master Prompt Template (คัดลอกข้อความด้านล่างนี้)

```text
Context & Goal: ฉันต้องการสร้าง Mod ภาษาไทยระดับมืออาชีพสำหรับเกม: [ชื่อเกม เช่น Dead Island 1] โดยเราจะทำงานร่วมกันทีละขั้นตอนอย่างเป็นระบบ นี่คือข้อมูลเบื้องต้นของโปรเจค:

Environment:
Workspace: [ตำแหน่งโฟลเดอร์ทำงาน เช่น E:\Mod_Workspace\DIDE\]
Game Directory: [ตำแหน่งติดตั้งเกม เช่น F:\SteamLibrary\steamapps\common\DIDE\]
Translation Tool: [ตำแหน่งสคริปต์แปลที่มี เช่น E:\Mod_Workspace\Tool\0_run_translation\run_translation_DL2_V2.py]

Milestones & Workflow (ขั้นตอนการทำงาน): โปรดรับทราบขั้นตอนการทำงานทั้งหมดด้านล่างนี้ เราจะทำไปทีละข้อ เมื่อเสร็จข้อหนึ่งฉันจะบอกให้ไปข้อถัดไป:

1. Extraction, Analysis & Failsafe (สกัดไฟล์และแบ็คอัพต้นฉบับ):
- ค้นหาและสกัดไฟล์ข้อความ (บทสนทนา, คัทซีน, UI, เมนู)
- กฎเหล็ก: ให้สร้างโฟลเดอร์ Backup_Original ใน Workspace และแบ็คอัพไฟล์ต้นฉบับทั้งหมดไว้ก่อนเสมอ เพื่อป้องกันไฟล์เสีย
- วิเคราะห์โครงสร้างไฟล์ (.pak, .arc, .bin ฯลฯ) ว่าต้องใช้ Tool อะไรในการ Unpack/Pack

2. Font & UI Architecture (เจาะลึกระบบฟอนต์):
- ตรวจสอบว่าเกมใช้ฟอนต์กี่ชนิด อยู่ที่ไฟล์ไหน รูปแบบอะไร (.ttf, .otf, Sprite)
- เตรียมความพร้อมสำหรับสร้างหรือแปลงฟอนต์เป็นภาษาไทย

3. Pre-Translation Glossary (วิเคราะห์และดึงคำศัพท์เฉพาะ):
- ก่อนเริ่มแปลจริง ให้สแกนไฟล์ข้อความทั้งหมดเพื่อดึง "ชื่อเฉพาะ" (ตัวละคร, สถานที่, ไอเทม, อาวุธ, ศัตรู) ออกมาลิสต์ให้ฉันดู
- นำลิสต์นี้มากำหนดเป็นคำศัพท์มาตรฐาน (GLOSSARY) เพื่อให้การแปลทั้งเกมสม่ำเสมอกันตั้งแต่ต้นจนจบ

4. Proof of Concept (ทดสอบแปลและฟอนต์หน้าเมนู):
- แปลข้อความเฉพาะส่วน "หน้าเมนูหลัก" ก่อน (โดยใช้หลักการ: แปลให้กระชับ เพื่อไม่ให้ข้อความล้นกรอบ UI)
- แพ็คฟอนต์ภาษาไทยและไฟล์เมนูเข้าเกม เพื่อทดสอบว่าฟอนต์แสดงผลได้ สระไม่ลอย ไม่เป็น "เต้าหู้" หากมีปัญหาต้องแก้ให้จบที่สเต็ปนี้

5. Script Customization & Safety Rules (ดัดแปลงสคริปต์แปลภาษา):
- ดัดแปลง Translation Tool ให้รองรับโครงสร้างเกมนี้ โดยเพิ่มคำสั่ง "ห้ามดัดแปลงแท็กและตัวแปร" (เช่น %s, {Item_Name}, <color=red>) อย่างเด็ดขาด เพื่อป้องกันเกมแครช
- สำคัญ: เมื่อเขียนสคริปต์เสร็จ "ให้หยุดรอก่อน" เพื่อให้ฉันใส่ API Key และอัปเดตไฟล์ GLOSSARY ก่อนรันแปลชุดใหญ่

6. Mass Translation & Automated QA (แปลชุดใหญ่และใช้สคริปต์ตรวจสอบ):
- เมื่อฉันอนุญาต ให้รันสคริปต์แปลไฟล์ทั้งหมด
- ก่อนทำการ Pack กลับเข้าเกม ให้คุณเขียนสคริปต์สั้นๆ (Validator Script) เพื่อเช็คเทียบไฟล์อังกฤษกับไทย ว่ามีโค้ดสี วงเล็บ หรือตัวแปร %s หายไปหรือไม่
- หากสคริปต์ไม่พบ Error จึงจะทำการแพ็คไฟล์ทั้งหมดเข้าเกมได้

7. Final Playtest (ตรวจสอบคุณภาพและจบโปรเจค):
- ฉันจะทดสอบเข้าเล่นจริงเพื่อเช็คบัค โค้ดรั่ว หรือข้อความล้นกรอบ หากสมบูรณ์แบบแล้วจึงจบโปรเจค

Instructions for AI: ตอนนี้ให้คุณตอบรับว่าเข้าใจเป้าหมายและขั้นตอนทั้งหมดอย่างละเอียด และให้เริ่มทำงานใน ขั้นตอนที่ 1 ได้เลย (บอกฉันว่าคุณจะเริ่มแบ็คอัพที่โฟลเดอร์ไหน และจะสกัดไฟล์อย่างไร)
```

---

> [!NOTE] ทำไมต้องใช้ Prompt นี้?
> การเขียนเป้าหมาย 7 ข้อล่วงหน้าให้ AI รับรู้ จะทำให้บอทไม่พยายามทำงานข้ามขั้นตอน (เช่น พยายามแปลไฟล์ทั้งเกมในแชทเดียว ซึ่งจะทำให้บอทหลอนหรือ Error อย่างแน่นอน) แต่บอทจะคอยถามความคืบหน้าคุณทีละขั้นตามแผนผังที่วางไว้ครับ
