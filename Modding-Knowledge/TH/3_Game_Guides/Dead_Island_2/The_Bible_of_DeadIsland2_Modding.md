# คัมภีร์วิชา: เจาะระบบม็อดภาษาไทย Dead Island 2 (The Modding Bible)

คัมภีร์ฉบับนี้ (Knowledge Item - KI) รวบรวม **"เทคนิคขั้นสูง"** ที่ใช้ในการทะลวงโครงสร้างของเกม Dead Island 2 (Unreal Engine 4.27) เพื่อบังคับให้เกมสามารถแสดงผลภาษาไทยได้สมบูรณ์แบบ ถือเป็นกรณีศึกษา (Case Study) สำหรับการทำม็อดเกมสมัยใหม่ที่ใช้ระบบแพ็คไฟล์แบบ **IoStore**

---

## 1. ปัญหาของเกมสมัยใหม่ (IoStore System)
เกมที่พัฒนาด้วย Unreal Engine 4 รุ่นหลังๆ (รวมถึง UE5) จะเลิกใช้ระบบ `.pak` ธรรมดาในการเก็บไฟล์เกมหลัก แต่เปลี่ยนมาใช้ระบบ **IoStore** ซึ่งประกอบไปด้วยไฟล์สองชนิดคือ:
- **`.utoc` (Table of Contents):** สารบัญที่เก็บที่อยู่ของไฟล์ (Offset/Length) และรหัส Chunk ID
- **`.ucas` (Container):** ก้อนข้อมูลยักษ์ (มักจะใหญ่ถึง 3-5 GB) ที่บีบอัดและเข้ารหัสไฟล์ทั้งหมดเอาไว้ด้วย **AES Encryption**

**ปัญหา:** การใส่ไฟล์ฟอนต์ภาษาไทยนามสกุล `.uasset` ลงไปในโฟลเดอร์ `~mods` ตามวิธีคลาสสิก **"จะใช้ไม่ได้ผล"** เพราะเอนจิ้นจะดึงฟอนต์หลักจากระบบ IoStore ก่อนเสมอ ทำให้ผู้เล่นเห็นสี่เหลี่ยม (Tofu) แทนภาษาไทย

---

## 2. เทคนิคที่ 1: การผ่าตัดฝังฟอนต์ลง IoStore (Direct UCAS Injection)
เพื่อแก้ปัญหาเกมไม่อ่านฟอนต์ไทย เราต้องทำการแฮ็กไฟล์ระบบของเกมโดยตรงด้วย Python โดยใช้เทคนิคดังนี้:

1. **ถอดรหัส AES:** ต้องรู้ AES Key ของเกม Dead Island 2 เพื่อให้สามารถอ่านโครงสร้างได้
2. **คำนวณ Chunk ID:** แปลงชื่อ Path ของฟอนต์ (เช่น `DeadIsland/Content/DI2/UI/Fonts/fonts_en`) ให้กลายเป็น Hash (12 Bytes) เพื่อค้นหาว่าสารบัญใน `.utoc` ชี้ไปที่บรรทัดไหน
3. **Append (ต่อท้ายไฟล์):** แทนที่เราจะเขียนทับฟอนต์เดิมใน `.ucas` ซึ่งเสี่ยงทำให้ข้อมูลพัง เราใช้วิธี **"นำไฟล์ฟอนต์ไทย (`fonts_en.uasset`) ไปแปะต่อท้ายสุดของไฟล์ `pakchunk0-WindowsNoEditor.ucas`"**
4. **Rewrite Offset:** กลับไปแก้ไขตัวเลขในสารบัญ (`.utoc`) โดยเปลี่ยนที่อยู่ (Offset) ของฟอนต์เดิม ให้ชี้กระโดดมาที่ **"ท้ายไฟล์"** ที่เราเพิ่งแปะฟอนต์ไทยลงไปแทน 

*เทคนิคนี้ทำให้เราไม่ต้องแจกไฟล์ `.ucas` ขนาด 3GB ให้ผู้เล่นโหลด แต่แจกแค่โค้ด Python ขนาดไม่กี่ KB ที่ไปผ่าตัดไฟล์ของผู้เล่นเองในเครื่อง*

## 3. เทคนิคที่ 2: ปลอมแปลงลายเซ็นดิจิทัล (Signature Bypass)
แม้เราจะฝังฟอนต์สำเร็จแล้ว แต่ข้อความภาษาไทยที่เราแปล (เก็บในไฟล์ `Game.locres` และแพ็คเป็น `.pak`) จะถูกเกมปฏิเสธไม่ยอมโหลดอยู่ดี เนื่องจากเอนจิ้นมีระบบตรวจสอบลายเซ็น **Signature Check** 

**วิธีแก้ปัญหา:**
- ระบบตรวจจับของ UE4 จะมองหาไฟล์นามสกุล `.sig` ที่ชื่อตรงกับไฟล์ `.pak`
- เราใช้วิธี **"ขโมย"** ไฟล์ `.sig` ดั้งเดิมของเกม (เช่น `pakchunk_default-WindowsNoEditor_P.sig`) มาทำการ **Rename** ให้ชื่อตรงกับไฟล์ม็อดของเรา (เช่น `pakchunk_default-WindowsNoEditor_Thai_P.sig`)
- เพียงเท่านี้ เอนจิ้นก็จะโดนหลอกว่าม็อดภาษาไทยของเราเป็นแพตช์อัปเดตอย่างเป็นทางการจากผู้พัฒนา และยอมโหลดภาษาไทยขึ้นมาแสดงผล

---

## 4. เทคนิคที่ 3: กระบวนการจัดการไฟล์แปล (Localization Pipeline)
การทำให้ภาษาไทยแสดงในเกมได้สมบูรณ์ ต้องมีการจัดการประเภทไฟล์ข้อความ 2 รูปแบบที่ต่างกัน:

1. **ระบบ UI และไอเทม (`.locres`)**
   - ข้อความทั่วไปของเกมถูกบีบอัดเป็น Binary ไฟล์นามสกุล `.locres`
   - เทคนิค: ใช้โปรแกรม `pylocres` แตกไฟล์ออกมาเป็น `.csv` -> นำไปแปลด้วย AI (ทำ Masking ซ่อนโค้ดสีและตัวแปร) -> แล้วใช้ `pylocres` บีบกลับเป็น `.locres` ตามเดิม
2. **ระบบซับไตเติลบทสนทนา (`.xml`)**
   - ซับไตเติลทั้งหมดของ Dead Island 2 ถูกฝังอยู่ในโครงสร้าง Tag ของไฟล์ `DialogueList.xml`
   - เทคนิค: เขียนสคริปต์ Python ใช้ไลบรารี `xml.etree.ElementTree` ในการค้นหา Node ต้นฉบับ และแทรก Node `<String>` ที่เป็นภาษาไทยเข้าไปขนาบข้างภาษาอังกฤษ ทำให้เกมสามารถดึงซับไทยขึ้นมาโชว์ได้

---

## 5. เทคนิคที่ 4: การแพ็คไฟล์กลับเข้าเกม (Repacking via UnrealPak)
เมื่อจัดการไฟล์แปลและซับไตเติลเสร็จแล้ว ไฟล์ทั้งหมดจะถูกนำไปจัดโครงสร้างโฟลเดอร์ให้ตรงกับต้นฉบับ (เช่น `DeadIsland/Content/Localization/...`) จากนั้นต้องทำการมัดรวมเป็นไฟล์ `.pak`

- **เครื่องมือที่ใช้:** `UnrealPak.exe` (เครื่องมือมาตรฐานของ Unreal Engine)
- **การสร้าง Response File:** แทนที่จะพิมพ์คำสั่งที่ยาวและซับซ้อน เราใช้สคริปต์ Python (`pack_mod.py`) ในการกวาดรายชื่อไฟล์ทั้งหมดในโฟลเดอร์ แล้วสร้างเป็นไฟล์ `response.txt` ที่ระบุที่อยู่ไฟล์ต้นทางและปลายทาง (Mount Point) อย่างแม่นยำ
- **การบีบอัด:** สั่งรัน `UnrealPak.exe` โดยดึงข้อมูลจาก `response.txt` พร้อมพารามิเตอร์ `-compress` เพื่อให้ไฟล์ม็อดถูกบีบอัดจนมีขนาดเล็กที่สุด
- **Naming Convention:** เพื่อให้เกมยอมโหลดไฟล์ม็อดไปทับไฟล์ดั้งเดิม ไฟล์แพ็คที่ได้จะต้องตั้งชื่อโดยลงท้ายด้วย `_P` (Patch Chunk) เสมอ เช่น `DeadIsland2_TH_P.pak`

---

## 6. เทคนิคที่ 5: การสร้าง Standalone Smart Installer
เนื่องจากกระบวนการ Inject ไฟล์ `.ucas` มีความซับซ้อนและผู้เล่นทั่วไปทำเองไม่ได้ เราจึงใช้เทคนิค **PyInstaller**

- **ฝังไฟล์ลงตัวรัน:** ใช้ฟังก์ชัน `--add-data` ของ PyInstaller ในการอัดไฟล์ฟอนต์ไทย, ไฟล์ `.pak` ของข้อความแปล, และสคริปต์ผ่าตัด รวมกันกลายเป็นไฟล์ `.exe` ตัวเดียว (Standalone)
- **Auto-Detect:** เขียนให้สคริปต์สามารถค้นหาตำแหน่งโฟลเดอร์ของ Dead Island 2 ในเครื่องผู้เล่นได้เอง (รองรับทั้ง Steam และ Epic)
- **ผลลัพธ์:** เปลี่ยนกระบวนการแฮ็กที่ซับซ้อน ให้กลายเป็นการกด Double Click เพียงครั้งเดียวของผู้ใช้งาน
