# 📘 คัมภีร์การสร้าง Mod ภาษาไทย Dying Light 2 (The Modding Bible)

เอกสารฉบับนี้รวบรวมองค์ความรู้ทั้งหมดที่ใช้ในการสร้าง Mod ภาษาไทยสำหรับเกม Dying Light 2 (C-Engine) ที่ประสบความสำเร็จแบบ 100% โดยแก้ปัญหาตัวอักษรภาษาอังกฤษพัง และปัญหาสระลอย/สระจมได้อย่างสมบูรณ์

---

## ภาคที่ 1: การจัดการระบบฟอนต์ (Font Modding)

ปัญหาหลักของ C-Engine คือ **ไม่รองรับการแสดงผลสระภาษาไทยที่ซ้อนทับกัน** (เช่น สระอิ, ไม้เอก, ไม้โท) หากปล่อยไว้ตามปกติ สระจะลอย หรือทับกันจนอ่านไม่ออก นอกจากนี้ การใช้ Mod เก่า (เช่น NooneTranslator) มีการใช้วิธี **"เขียนทับ"** (Hijack) ภาพฟอนต์ภาษาอังกฤษเดิม ส่งผลให้ตัวอักษรภาษาอังกฤษบางตัว (เช่น `%`, วงเล็บ) กลายเป็นภาษาไทยแทน

### 1. การสร้างภาพ Atlas (Texture) ใหม่
**ห้ามเขียนทับฟอนต์เดิมของเกมเด็ดขาด** วิธีที่ถูกต้องคือ:
1. ใช้สคริปต์ Python (`Pillow`) สร้างภาพขนาด `4096x4096` ขึ้นมาใหม่เอี่ยม
2. วาดตัวอักษรภาษาอังกฤษที่ถูกต้องทั้งหมดลงไป
3. วาดตัวอักษรภาษาไทยพื้นฐาน
4. วาด **"ตัวอักษร PUA" (Private Use Area)** ซึ่งเป็นตัวอักษรที่เราผสมสระเสร็จแล้ว (เช่น ด + ํ = ดํ) แล้วนำไปวางไว้ในช่องว่างของตาราง Unicode (เช่น `F269`)
5. เซฟภาพและแปลงเป็นไฟล์ `.dds` (DirectDraw Surface) รูปแบบ `BC3 (DXT5)`
6. นำภาพ `.dds` ไปแพ็คลงในไฟล์ `gui_common_pc.rpack` ด้วยเครื่องมือแพ็คของเกม

### 2. การแก้สคริปต์ UI ของเกม (.scr)
เพื่อให้เกมรับรู้ว่าเราวาดภาพฟอนต์ใหม่ไว้ที่ไหน เราต้องไปแก้ไฟล์โครงสร้าง UI ใน `data0.pak`:
- **`gui_default_font.scr` และ `vanilla.scr`**: 
  - สั่งให้เกมสร้าง `Material` ใหม่ โดยชี้พิกัดไปที่ไฟล์ `.dds` ตัวใหม่ที่เราเพิ่งสร้าง (เช่น สมมติให้เป็น `common_fonts_7`)
  - **ห้าม** ให้มันชี้ไปที่ `common_fonts_0` เดิม เพราะนั่นคือการทำให้ฟอนต์ชนกัน
- **`font_replacements.scr`**:
  - เป็นไฟล์ที่กำหนดว่า Unicode ตัวไหน อยู่พิกัด (UV) ไหนบนภาพ 4096x4096
  - เราต้องระบุพิกัดของตัวอักษรภาษาอังกฤษปกติ, ภาษาไทยปกติ, และ **ตัวอักษร PUA (เริ่มตั้งแต่ F000 ขึ้นไป)** ให้ตรงกับภาพที่เราวาดเป๊ะๆ

## ภาคที่ 2: ระบบจัดการคำแปลและ PUA (Translation & PUA System)

เมื่อเรามีฟอนต์ที่มีสระครบสมบูรณ์แบบ (ในรูปแบบ PUA) แล้ว เราต้องเขียนสคริปต์เพื่อแปลงข้อความภาษาไทยธรรมดา ให้กลายเป็นรหัส PUA ก่อนยัดเข้าเกม

### 1. การถอดรหัสข้อความเกม (Binsloc)
เกมเก็บข้อความทั้งหมดไว้ในไฟล์ `texts/pc/pc_en_lang.binsloc` ซึ่งถูกบีบอัดอยู่ใน `dataen.pak`
- เราใช้ `binsloc_tool.py unpack` ในการถอดรหัสออกมาเป็นไฟล์ `.csv`
- นำไฟล์ `.csv` นี้ไปใช้ในโปรแกรม **DyingLight2_Thai_Studio** เพื่อให้หน้าต่าง UI ขึ้นมาให้คนแปลสะดวกๆ

### 2. กระบวนการแปลงตัวอักษร (PUA Conversion)
นี่คือหัวใจสำคัญในการทำให้สระไม่ลอย:
1. สคริปต์ `Build_DL2_Mod.py` จะรับไฟล์ `.csv` ที่แปลเป็นภาษาไทยปกติมา
2. สคริปต์จะอ่านไฟล์ `mapping.json` (พจนานุกรม PUA) 
3. มันจะค้นหาคำที่มีสระซ้อนทับกัน (เช่น `ดำ`, `กั่`) แล้วแทนที่ตัวอักษรชุดนั้นด้วย **"รหัส PUA"** ที่เราวาดไว้ในภาคที่ 1 (เช่น เปลี่ยน `ดำ` เป็น `\uf269`)
4. **⚠️ ทริคสำคัญ (สระอำ):** เอนจิ้นของเกมมีบัคในการวาดสระอำ (`ำ`) โดยมันมักจะวาดแค่ "วงกลมด้านบน" (`ํ` นิคหิต) ทิ้งไว้ เราจึงต้องเขียนโค้ดบังคับต่อท้ายสระอำด้วยสระอาเสมอ (`text = text.replace('ำ', 'ำา')`) เพื่อให้มันแสดงผลคำว่า "ทำ" ได้สมบูรณ์ ไม่ใช่ "ท ํ"

### 3. การแพ็คไฟล์กลับเข้าเกม (Deployment)
เมื่อแปลงข้อความ PUA เสร็จสิ้น สคริปต์จะแพ็คไฟล์ `.csv` กลับเป็น `.binsloc`
- **⚠️ ข้อควรระวังขั้นสุดยอด:** การยัดไฟล์ `.binsloc` เข้าไปใน `dataen.pak` (ซึ่งคือไฟล์ ZIP) **ห้ามใช้โหมด Append ('a')** บนไฟล์เก่าเด็ดขาด!
- การ Append จะทำให้มีไฟล์ชื่อ `pc_en_lang.binsloc` **ซ้ำกัน 2 ไฟล์** ใน ZIP เดียว!
- เอนจิ้นเกม C-Engine เป็นระบบที่ไม่อ่านไฟล์ ZIP แบบปกติ มันจะเลือกอ่านไฟล์ที่อยู่ "บนสุด" เสมอ ทำให้ถ้ามีไฟล์ซ้ำ มันจะไปหยิบภาษาอังกฤษตัวเก่ามาแสดงผล และพอภาษาอังกฤษตัวเก่า (ที่ไม่ได้ทำ PUA) มาเจอกับฟอนต์ PUA ของเรา มันจะแสดงผลออกมาเป็นตัวประหลาด (เช่น **"ด๋าเนิน"**)
- **การแก้ปัญหา:** ต้องล้างสร้างไฟล์ ZIP (`.pak`) ใหม่ทุกครั้ง โดยคัดลอกไฟล์เก่ามาทั้งหมด ยกเว้นไฟล์ `.binsloc` แล้วค่อยยัด `.binsloc` ตัวใหม่เข้าไปปิดท้าย

## ภาคที่ 3: สรุปโฟลเดอร์สำหรับทำงานจริง (Workspace Structure)

เพื่อให้การทำงานเป็นระบบและ Standalone เราได้รวมทุกอย่างไว้ดังนี้:
- **`01_Releases`**: โฟลเดอร์ที่เก็บ `data0.pak` และ `dataen.pak` ที่สมบูรณ์พร้อมอัปโหลดแจกผู้เล่น
- **`128_Base`**: สถานที่ศักดิ์สิทธิ์สำหรับ "สร้างและแพ็คฟอนต์" (ทำแค่ตอนอยากเปลี่ยนลายฟอนต์)
- **`DyingLight2_Thai_Studio`**: โปรแกรมการแปลที่เชื่อมโยงกับ `Deploy_Mod.py` แปลเสร็จ -> กดเซฟ -> ดับเบิลคลิก Deploy -> ไฟล์วิ่งเข้าเกมพร้อมทดสอบทันที!
