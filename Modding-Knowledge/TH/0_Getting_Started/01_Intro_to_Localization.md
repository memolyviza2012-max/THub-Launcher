# 📖 บทที่ 1: ปฐมนิเทศนักม็อด (Introduction to Game Localization)

ยินดีต้อนรับเข้าสู่วงการแปลเกม (Game Localization)! การเปลี่ยนภาษาในเกมไม่ใช่แค่การเอาไฟล์มาแปลด้วย Google Translate แล้วนำไปวางทับ แต่มันคือการทำ **วิศวกรรมย้อนกลับ (Reverse Engineering)** เพื่อค้นหาว่าข้อความถูกซ่อนอยู่ที่ไหน

## 1. ข้อความในเกมซ่อนอยู่ที่ไหน?
เกมแต่ละเกมถูกสร้างมาด้วยเครื่องยนต์ (Engine) ที่ต่างกัน ทำให้วิธีการเก็บข้อความต่างกันไป:
- **แบบใจดี:** ข้อความถูกเก็บไว้ในโฟลเดอร์เกมตรงๆ ด้วยนามสกุลไฟล์ที่คนอ่านออก เช่น `.json`, `.xml`, `.csv`, `.ini`, `.yaml` ไฟล์พวกนี้คุณสามารถเปิดด้วย Notepad หรือ VSCode เพื่อแปลได้ทันที!
- **แบบสายแข็ง:** ข้อความถูกบีบอัดรวมกับก้อนโมเดลและรูปภาพ กลายเป็นไฟล์ยักษ์ๆ เช่น `.pak`, `.arc`, `.upk`, `.ba2` คุณจำเป็นต้องใช้โปรแกรมเฉพาะทาง (Extractor) ในการ "แตกไฟล์" ออกมาก่อน
- **แบบฮาร์ดคอร์:** ข้อความถูกฝังลึกอยู่ในไฟล์โค้ดของเกมอย่าง `.exe` หรือ `.dll` โดยตรง ซึ่งแบบนี้ต้องใช้เทคนิคขั้นสูง (Memory Hook) เพื่อดึงข้อความออกมาระหว่างที่เกมกำลังรัน

## 2. กระบวนการแปลเกมมาตรฐาน (Standard Workflow)
ไม่ว่าเกมนั้นจะง่ายหรือยาก กระบวนการทำงานจะคล้ายกันเสมอคือ:
1. **Unpack (แตกไฟล์):** ใช้เครื่องมือเจาะเอาไฟล์ข้อความออกมาจากก้อนตัวเกม
2. **Translate (แปล):** นำไฟล์นั้นมาแปล (จะแปลมือหรือใช้ AI TStudio/TRun ของเราก็ได้)
3. **Font Creation (สร้างฟอนต์):** เปลี่ยนรูปภาพตัวอักษรในเกมให้มีสระและพยัญชนะไทย
4. **Pack (แพ็กกลับ):** บีบอัดไฟล์แปลและฟอนต์ กลับเข้าไปในรูปแบบเดิมที่เกมรู้จัก
5. **Playtest (ทดสอบ):** เข้าเกมเพื่อดูว่าสระลอยไหม ข้อความล้นกรอบหรือเปล่า
