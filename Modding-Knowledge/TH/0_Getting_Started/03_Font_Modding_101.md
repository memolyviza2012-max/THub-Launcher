# 🔠 บทที่ 3: พื้นฐานฟอนต์ภาษาไทย (Font Modding 101)

ปัญหาที่ทำให้โปรเจกต์แปลเกม 90% ต้องล้มเลิกไป ไม่ใช่เพราะแปลไม่ไหว แต่เพราะ "ทำฟอนต์ภาษาไทยเข้าเกมไม่ได้" หรือที่เรียกกันว่าอาการ "ตัวเต้าหู้ (สี่เหลี่ยม)" และ "สระลอย"

## 1. รูปภาพฟอนต์ (Font Atlas) คืออะไร?
เอนจินเกมส่วนใหญ่ไม่ได้ดึงฟอนต์ `.ttf` จาก Windows มาใช้ตรงๆ แต่มันจะวาดตัวอักษรทุกตัวลงบน "รูปภาพขนาดใหญ่" (Texture Atlas) และเกมจะคำนวณพิกัด (X, Y) เพื่อตัดเอารูปตัวอักษรไปแปะบนหน้าจอทีละตัว
- **SDF (Signed Distance Field):** คือเทคนิคฟอนต์ภาพยุคใหม่ ที่ทำให้ขอบตัวอักษรเบลอๆ เวลาเกมนำไปขยายมันจะคมชัดเสมอโดยพิกเซลไม่แตก

## 2. ทำไมสระถึงลอย/จม?
เกมฝรั่งถูกออกแบบมาให้พิมพ์ข้อความเรียงกันจากซ้ายไปขวา (ก ข ค) แต่ภาษาไทยมีลักษณะการพิมพ์แบบ "ซ้อนทับแนวตั้ง" (เช่น คำว่า "ที่" มีทั้งพยัญชนะ สระบน และวรรณยุกต์ซ้อนกัน 3 ชั้น) 
เมื่อเกมฝรั่งพยายามพิมพ์ภาษาไทย มันจึงเอาสระไปวางไว้ด้านขวาของพยัญชนะ (ก ิ) แทนที่จะวางซ้อนไว้ด้านบน ทำให้สระลอยกระจัดกระจาย

## 3. เทคนิคพื้นที่ลับ PUA (Private Use Area)
เพื่อแก้ปัญหาสระลอย วงการม็อดชาวไทยจึงคิดค้นเทคนิคที่เรียกว่า **PUA**
ในระบบรหัสตัวอักษรคอมพิวเตอร์ (Unicode) จะมีพื้นที่ว่างโซนหนึ่งที่ชื่อว่า PUA (ตั้งแต่รหัส `E000` เป็นต้นไป) ซึ่งเป็นพื้นที่เปล่าๆ ไม่มีภาษาใดจับจอง
- **หลักการทำงาน:** เราจะเขียนสคริปต์เพื่อยุบรวม "พยัญชนะ+สระ+วรรณยุกต์" ให้กลายเป็นก้อนเดียวกัน (ตัวอักษรตัวเดียว) แล้วนำรูปก้อนนั้นไปแอบไว้ในพื้นที่ PUA 
- **ผลลัพธ์:** เมื่อเกมอ่านเจอคำว่า "ที่" มันจะไม่พิมพ์ `ท` + `สระอี` + `ไม้เอก` แต่สคริปต์ของเราจะสลับให้มันไปดึงรูปภาพรหัส `E001` (รูปที่ประกอบสำเร็จแล้ว) ขึ้นมาแสดงผลแทน ทำให้สระไม่ลอยเป๊ะ 100%

> [!TIP]
> เครื่องมือ **TPUA** และ **TGlyph** ใน THub มีหน้าที่จัดการระบบ PUA และวาดรูปภาพ Font Atlas ให้คุณอัตโนมัติ!
