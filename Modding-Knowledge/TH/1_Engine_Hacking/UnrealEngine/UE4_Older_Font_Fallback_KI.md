# 📚 Knowledge Item (KI): เทคนิคการทำภาษาไทยสำหรับ Unreal Engine 4 รุ่นเก่า (UE 4.14 และใกล้เคียง)

**โปรเจคที่ค้นพบ:** Space Hulk: Deathwing - Enhanced Edition
**เวอร์ชัน Engine:** Unreal Engine 4.14.3
**ปัญหาที่พบ:** เกมรุ่นเก่าไม่รองรับภาษาไทยในตัว การพยายาม Cook ฟอนต์ `.uasset` ผ่าน UE4 Editor รุ่นเก่าทำได้ยาก มักจะมีปัญหา Engine Crash หรือโครงสร้างไฟล์ไม่ตรงกัน และตัวเกมไม่ยอมอ่านไฟล์ม็อดจากโฟลเดอร์ `~mods`

---

## 🛠️ เทคนิคที่ 1: การใช้ฟอนต์ตัวแทน (Slate Font Fallback Trick)

เทคนิคนี้คือ "ท่าไม้ตาย" ในการแทรกภาษาไทยเข้าไปในเกม UE4 รุ่นเก่า โดยไม่ต้องแตะต้องไฟล์ฟอนต์ดั้งเดิมของเกมเลยแม้แต่ไฟล์เดียว

### 💡 หลักการทำงาน:
Unreal Engine 4 ทุกเวอร์ชันจะมีระบบฟอนต์ฉุกเฉินที่เรียกว่า "Slate Font Fallback" ซึ่งเอนจิ้นจะเรียกใช้ทันทีเมื่อไฟล์ฟอนต์หลักของเกมไม่รองรับตัวอักษรนั้นๆ (เช่น ตัวอักษรภาษาไทย) โดยค่าเริ่มต้น เอนจิ้นจะมองหาไฟล์ที่ชื่อว่า `DroidSansFallback.ttf` 

### 📝 วิธีทำ:
1. หาไฟล์ฟอนต์ที่รองรับภาษาไทย (เช่น Tahoma, DB Helvethaica) และเปลี่ยนชื่อไฟล์เป็น **`DroidSansFallback.ttf`**
2. สร้างโครงสร้างโฟลเดอร์สำหรับแพ็คไฟล์ม็อดดังนี้:
   ```text
   Pack/
   └── Engine/
       └── Content/
           └── Slate/
               └── Fonts/
                   └── DroidSansFallback.ttf
   ```
3. นำโฟลเดอร์ `Engine` นี้ไปแพ็ครวมกับไฟล์แปลภาษา (`.locres`) ของเราได้เลย
4. **ผลลัพธ์:** ตัวอักษรภาษาอังกฤษจะแสดงฟอนต์สวยงามตามปกติของเกม แต่เมื่อเกมเจอข้อความภาษาไทย มันจะดึงฟอนต์ `DroidSansFallback.ttf` ของเรามาแสดงผลแทนทันที!

## 🛠️ เทคนิคที่ 2: บังคับเกมให้อ่านไฟล์ม็อดด้วยสกุล `_P.pak` (Patch File)

เกม Unreal Engine รุ่นเก่าๆ (เช่น 4.14) **ยังไม่มีระบบรองรับโฟลเดอร์ `~mods`** ดังนั้นการนำไฟล์ม็อดไปโยนใส่โฟลเดอร์ `Paks` หรือ `~mods` ดื้อๆ เกมจะไม่ยอมโหลดและแสดงภาษาอังกฤษเหมือนเดิม

### 💡 หลักการทำงาน:
UE4 มีระบบอ่านไฟล์แพทช์อัปเดต (Patching System) โดยตัวเอนจิ้นจะถูกเขียนโค้ดมาให้โหลดไฟล์ `.pak` ที่มีชื่อลงท้ายด้วย `_P` เป็นลำดับสุดท้ายเสมอ เพื่อให้ไฟล์นั้นสามารถ "เขียนทับ (Override)" ไฟล์เกมหลักได้

### 📝 วิธีทำ:
1. เมื่อใช้โปรแกรม `repak` บีบอัดไฟล์เสร็จสิ้น (ต้องใช้ `--version V3` สำหรับ UE 4.14)
2. ให้ทำการ **เปลี่ยนชื่อไฟล์ .pak ของม็อดให้ลงท้ายด้วย `_P` เสมอ**
   - ❌ *ผิด:* `SpaceHulkGame-WindowsNoEditor_TH.pak` (เกมจะไม่โหลด)
   - ✅ *ถูก:* **`SpaceHulkGame-WindowsNoEditor_P.pak`** (เกมโหลดทับ 100%)
3. นำไฟล์ `_P.pak` ไปวางไว้คู่กับไฟล์ `.pak` ต้นฉบับของเกม ในโฟลเดอร์ `Content/Paks/` โดยไม่ต้องสร้างโฟลเดอร์ย่อย

## 🛠️ เทคนิคที่ 3: คำสั่งบรรจุไฟล์ (Packing Command) สำหรับ UE 4.14

เพื่อให้แน่ใจว่าไฟล์ `.pak` ที่ถูกสร้างขึ้นมาจะมีโครงสร้างดั้งเดิมที่เกมอ่านได้ (Legacy V3) เราต้องเจาะจงเวอร์ชันตอนใช้เครื่องมือ `repak` ด้วยคำสั่งผ่าน Command Line ดังนี้:

### 📝 วิธีแพ็ค:
1. เปิด Command Prompt หรือสร้างไฟล์ `.bat`
2. ใช้คำสั่งนี้ในการแพ็คโฟลเดอร์ `Pack` ของเรา:
   ```cmd
   repak.exe pack Pack SpaceHulkGame-WindowsNoEditor_P.pak --version V3
   ```
   *คำอธิบาย:* 
   - `pack`: คำสั่งให้บีบอัด
   - `Pack`: โฟลเดอร์ต้นทางที่เก็บโครงสร้างโฟลเดอร์ม็อดทั้งหมด
   - `SpaceHulkGame-WindowsNoEditor_P.pak`: ชื่อไฟล์ปลายทางที่ตั้งเป็น Patch ทันที
   - `--version V3`: **บังคับ** ให้ใช้แพทเทิร์นไฟล์ Pak แบบเก่าที่ UE 4.14 รองรับ

## 📌 บทสรุป (Summary)
หากต้องรับมือกับการแปลเกมตระกูล Unreal Engine 4 รุ่นบุกเบิก (ต่ำกว่า 4.19) ให้ใช้สูตรสำเร็จนี้เสมอ:
1. แอบยัดฟอนต์ `DroidSansFallback.ttf` ลงใน `Engine/Content/Slate/Fonts/`
2. แพ็คไฟล์ให้ตรงตามเวอร์ชันดั้งเดิม (เช่น `--version V3`)
3. เปลี่ยนชื่อไฟล์ม็อดให้ลงท้ายด้วย `_P.pak` แล้ววางทับในโฟลเดอร์ Paks ตรงๆ
