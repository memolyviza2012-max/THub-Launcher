# องค์ความรู้: การสร้าง Mod ภาษาไทยสำหรับ Mass Effect: Andromeda (Frostbite Engine)

เอกสารฉบับนี้สรุปเทคนิคและกระบวนการทางวิศวกรรม (Reverse Engineering) ทั้งหมดที่ใช้ในการเจาะระบบเกมและแก้ไขบั๊กอาการ "จอดำ" หรือ "เกมค้าง" เพื่อให้เกมบนเอนจิ้น Frostbite สามารถแสดงผลภาษาไทยและตัดคำได้อย่างสมบูรณ์แบบ 100%

---

## 1. ปัญหาของ Frostbite Engine กับภาษาไทย

ตัวเกมไม่ได้ถูกโปรแกรมมารองรับอักขระในช่วง Unicode ของภาษาไทย (`U+0E00` - `U+0E7F`) เมื่อเราพยายามยัดภาษาไทยเข้าไปตรงๆ จะเกิดปัญหาดังนี้:
1. **Memory Overflow:** เอนจิ้นไม่มีข้อมูล Kerning/Metrics ของอักขระไทยในระบบ ส่งผลให้อ่านค่าหน่วยความจำผิดพลาดและเกมค้าง (Crash)
2. **Word-Wrap Bug:** ภาษาไทยไม่มีการเว้นวรรคระหว่างคำ ระบบตัดคำ (Line-breaking) ของเกมจะพยายามหาช่องว่างเพื่อขึ้นบรรทัดใหม่ เมื่อหาไม่เจอ มันจะเกิดลูปอนันต์ หรือตัดคำกลางสระ (เช่น ตัดกลาง ก.ไก่ กับ สระ ำ) ทำให้สระลอยและเกมค้างเมื่อประโยคยาวเกินไป

---

## 2. เทคนิคการแก้ไข (The Solutions)

เพื่อก้าวข้ามข้อจำกัดของ Engine เราได้ใช้เทคนิค 4 ขั้นตอนหลักดังนี้:

### 💡 เทคนิคที่ 1: Glyph Swapping สู่ฐานภาษาจีน (CJK)
แทนที่จะใช้ภาษาไทยตรงๆ เราทำการ **หลอกเกม** ว่ากำลังแสดงผลภาษาอื่นที่เกมรองรับอยู่แล้ว
- **ทำไมต้องภาษาจีน?**: ในตอนแรกเราทดลองใช้ภาษารัสเซีย แต่พบว่ารัสเซียต้องการช่องว่างในการตัดคำ มิฉะนั้นเกมจะค้าง ส่วนภาษาญี่ปุ่นมีกฎ "Kinsoku Shori" (ห้ามเว้นวรรคหน้าเครื่องหมายวรรคตอน) ทำให้เกิดบั๊กตัดคำ เราจึงจบที่ **ภาษาจีน (CJK Unified Ideographs - `U+4E00`)** ซึ่งเกมอนุญาตให้ตัดคำตรงไหนก็ได้โดยไม่เกิดบั๊ก
- **วิธีทำ**: ใช้ไลบรารี `fonttools` ใน Python (`font_swapper.py`) เข้าไปแก้ไฟล์ฟอนต์ `.ttf` โดยคัดลอกรูปทรง (Glyph) ของอักษรไทยทั้งหมด ไปวางทับในช่อง Unicode ของภาษาจีน 

### 💡 เทคนิคที่ 2: การตัดคำอัจฉริยะ (Smart Word-Wrap ด้วย ZWSP)
เพื่อแก้ปัญหาที่เกมมักจะตัดคำแบบผิดๆ (เช่น ตัดพยัญชนะออกจากสระ) 
- เราใช้ **AI ตัดคำภาษาไทย (PyThaiNLP)** เข้ามาอ่านประโยคภาษาไทยทั้งหมด 104,000 ประโยค
- แทรก **ช่องว่างล่องหน (Zero-Width Space - `U+200B`)** เข้าไประหว่างคำ เช่น `กำหนด[ZWSP]ไว้`
- **การป้องกันโค้ดเกมพัง**: เราเขียน Regular Expression (Regex) ให้ AI เลือกสับเฉพาะ "บล็อกข้อความที่เป็นภาษาไทยแท้ๆ" เท่านั้น เพื่อป้องกันไม่ให้มันไปสับโค้ด HTML ของเกม (เช่น `<font color='#FF0000'>`) ซึ่งเป็นสาเหตุหลักที่ทำให้เกมค้างทันทีที่เข้าหน้าเมนู

### 💡 เทคนิคที่ 3: ยกสระและวรรณยุกต์ (Font Y-Shifting)
ฟอนต์ภาษาอังกฤษไม่ได้ออกแบบมาให้มีอักขระซ้อนกันบนล่างเหมือนภาษาไทย ทำให้สระบน (เช่น  ิ  ี) และวรรณยุกต์ (เช่น  ่  ้) จมลงไปเบียดกับพยัญชนะ
- เราใช้ Python (`shift_vowels.py`) เข้าไปแก้ไขพิกัด `Y-axis` ของสระและวรรณยุกต์เป้าหมายทั้งหมดในตาราง `glyf` ของไฟล์ `.ttf` โดยตรง
- ยกสระบนและวรรณยุกต์ขึ้น **50 Units (UPM)** ซึ่งเป็นระยะที่สวยงามและไม่ลอยจนเกินไป

### 💡 เทคนิคที่ 4: การเข้ารหัสระดับ Binary (Bypass Frosty Plugin)
เครื่องมือ Frosty Editor มีปลั๊กอิน `BiowareLocalizationPlugin` ที่มีข้อจำกัดในการรองรับ Mod ภาษาขนาดใหญ่ 
- เราทำการ Reverse Engineering โครงสร้างไฟล์ `.res` ของเกม (ซึ่งเข้ารหัสด้วย Huffman Tree และ 32-bit Pointers แบบ LSB-first)
- เขียน Script Python (`process_all_res.py`) เพื่อสร้างไฟล์ `.res` ขึ้นมาใหม่ด้วยตัวเอง 100% ทำให้สามารถฉีดข้อความภาษาไทยที่ถูกเข้ารหัสเป็นอักษรจีน+ช่องว่างล่องหน เข้าไปแทนที่ภาษาอังกฤษใน `.res` ทั้ง 61 ไฟล์ได้ภายในไม่กี่วินาที

## 3. เครื่องมือ (Tools) ที่สร้างขึ้น

เก็บไฟล์ Script เหล่านี้ไว้ใช้ในอนาคตได้เลยครับ (อยู่ที่ `Tools\Localization_Scripts\`):
1. **`process_all_res.py`**: สคริปต์หลักสำหรับประมวลผลข้อความจาก CSV -> ตัดคำด้วย AI -> แปลงเป็นอักษรจีน -> เข้ารหัสเป็นไฟล์ `.res` 
2. **`font_swapper.py`**: สคริปต์สำหรับนำฟอนต์ภาษาไทย ไปทับช่องภาษาจีนในไฟล์ `.ttf`
3. **`shift_vowels.py`**: สคริปต์สำหรับยกสระและวรรณยุกต์ให้สูงขึ้น
4. **`mea_loc_decoder.py`** / **`mea_loc_encoder.py`**: เครื่องมือสำหรับถอดรหัส/เข้ารหัสไฟล์ .res แบบแยกเดี่ยว เพื่อตรวจสอบข้อมูลภายใน

> [!TIP]
> **การประยุกต์ใช้ในอนาคต**
> องค์ความรู้นี้สามารถนำไปใช้แปลเกมอื่นๆ ของค่าย EA ที่ใช้ **Frostbite Engine** ได้เกือบทุกเกม (เช่น Dragon Age, Dead Space Remake, Battlefield) เพราะโครงสร้างของ Font Resource และ Localization Resource ใช้วิธีการเดียวกันทั้งหมดครับ!
